## 这是什么

面向海外智能门锁 AI 功能规划的 **全维度场景枚举工具**。

将「人物 × 用房类型 × 时刻 × 住宅类型 × 事件」进行叉乘组合（理论 **80,640** 种），每种组合可展开查看：

- 卡片上直接显示**最痛一条痛点**（不展开也能扫一遍）
- 展开后：人物画像 + 按**回家前/开门/门内/出门前**等时间节点拆解的详细旅程（行为/触点/心理活动/摩擦点）
- Moment of Truth
- 关联度评分（高/中/低）

## 维度来源

基于「北美本土生活门前场景枚举」表格：

| 维度 | 枚举数 |
|------|--------|
| 人物 | 14（常住人/访客/服务人员/物流） |
| 用房类型 | 5 |
| 时刻 | 8 |
| 住宅类型 | 8 |
| 事件 | 18（含通勤/遛狗/取包裹等高频个人触发 + 派对/节日/装修/搬家等低频家庭级背景事件） |

> v1.2 起，原「活动」维度（派对/节日/装修/搬家）已并入「事件」维度。两者本质都是"这次为什么要开这扇门"，拆成两个平行维度叉乘会产生大量不自然的组合（如「遛狗×装修」），故合并为一个维度，总组合数从 313,600 降至 80,640。

## 关联度说明

评分 = **时刻×事件** 基础分，再叠加多维惩罚/上限：

| 维度组合 | 示例（降权） |
|---------|-------------|
| 住宅×事件 | 度假屋+通勤、55++上学 |
| 用房×事件 | 空置/Airbnb+通勤上学、商用+育儿派对 |
| 用房×时刻 | 空置+居家/门内 |
| 人物×事件/住宅 | 外卖+通勤、子女+55+社区 |
| 硬性上限 | 度假屋通勤 ≤12、空置日常通勤 ≤18 |

- **≥70 高关联**：真实高频，优先规划
- **45–69 中关联**：会发生，可作二期
- **<45 低关联**：理论组合，现实中少见

默认显示关联度 ≥40 的场景（约 3.2 万条），可勾选「显示全部组合」。
